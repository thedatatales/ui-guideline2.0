Stepper Component

The stepper component provides a wizard-like workflow by dividing content into logical steps. It supports multiple variants including horizontal, vertical, compact progress, progress timeline, and card steppers. It also supports linear progression with form validation and custom step templates.

Basic Horizontal Stepper

Basic stepper displays steps horizontally. Each step is defined using bntv-step component with a stepLabel property.

Component Properties

Property Type Default Description
type 'horizontal' | 'vertical' | 'compactProgress' | 'progressTimeline' 'horizontal' Type of the stepper layout. Use 'compactProgress' for compact progress view, 'progressTimeline' for timeline view
isLinear boolean false Whether the validity of previous steps should be checked
labelPosition 'end' | 'bottom' 'bottom' Position of the step label
selectedIndex number 0 The index of the selected step
disableRipple boolean false Whether ripples should be disabled for step headers
stepLabel string - Plain text label of the step
stepDescription string - Description text for the step
stepFormGroup UntypedFormGroup - The top level abstract control of the step (for validation)
isEditable boolean true Whether the user can return to this step once completed
stepperSelectionChange EventEmitter - Event emitted when the selected step has changed
iconName string - Icon name for the step (used in compact progress stepper)
isStreched boolean false Whether steps should be stretched to fill available space
cardType 'normal' | 'fullScreen' 'normal' Card type for card stepper (bntv-card-stepper only)
heading any - Heading text for card step (bntv-step-card only)

Usage Guidelines

  • Wizard Workflows: Use stepper for multi-step forms, wizards, or processes that require sequential completion
  • Form Validation: Use linear stepper with form groups to enforce step-by-step validation
  • Layout: Choose horizontal layout for desktop and vertical for mobile or narrow spaces
  • Compact Progress: Use compact progress stepper when you need a space-efficient progress indicator with custom icons
  • Progress Timeline: Use progress timeline stepper for time-based workflows where you need to display timestamps and durations
  • Card Stepper: Use card stepper for card-based navigation interfaces where each step is displayed as a card
  • Step Labels: Provide clear, concise labels for each step
  • Step Descriptions: Add descriptions to provide additional context for each step, especially useful for timeline steppers
  • Icons: Use iconName property to add custom icons to steps in compact progress stepper
  • Navigation: Control step navigation programmatically using selectedIndex property
  • Custom Templates: Use stepLabelTemplateRef to customize step label appearance
  • Progress Indicators: Stepper automatically shows progress indicators for completed, current, and upcoming steps
