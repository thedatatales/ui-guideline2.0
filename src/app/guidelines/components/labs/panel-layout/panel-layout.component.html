Panel

Page-level content organizers. Panels structure, they do not explain content. Entity set is small and non-overlapping with cards. See migration-plan/PANEL_SYSTEM_REFERENCE.md. Styles live in panel-layout.component.scss for now; will move to theme later.

1. Canonical panel

Full structure: header (title, subtitle, description, actions), divider, body, footer (status, cta). Spacing uses --panel-padding, --panel-section-gap, --panel-header-gap.

Context or date range

Optional description. Panel does not care if this is plain text or HTML.

Body is layout-neutral. Tabs, tables, forms, cards, grids go here. Panel only regulates grouping.

3. Entity reference

Core structural and placement entities. Forbidden: __metric, __icon, __delta, __value, __kpi.

Entity Required Purpose
.vw-panel✓Root container
.vw-panel__headerOptionalTitle, subtitle, description, actions
.vw-panel__body✓Free-form content (layout-neutral)
.vw-panel__footerOptionalStatus, CTA, meta
.vw-panel__dividerOptionalSeparator between sections
.vw-panel__titleOptionalPrimary heading (header)
.vw-panel__subtitleOptionalContext (header)
.vw-panel__descriptionOptionalExplanatory text (header)
.vw-panel__actionsOptionalButtons/links (header)
.vw-panel__statusOptionalStatus text/pills (footer)
.vw-panel__ctaOptionalActions (footer)
.vw-panel__metaOptionalTimestamps, pagination (footer)
