Grid Component

The grid component provides a powerful data table with features like sorting, filtering, pagination, and server-side data loading. It's built on top of AG-Grid and supports both client-side and server-side data modes.

Basic

Basic grid component with client-side data. Use agGridConfig to provide column definitions and row data. Set agGridMode to 'clientSide' for client-side data handling.

Component Properties

Property Type Default Description
agGridName string - Unique identifier for the grid instance
agGridMode string 'serverSide' Grid mode: 'clientSide' or 'serverSide'
agGridConfig IGridConfig - Grid configuration object with columnDefs and gridOptions
agGridInitialApiCall boolean true Trigger initial API call for server-side grids
agGridModuleName string - Module name for grid context
type string 'grid' Grid type: 'grid', 'kanban', 'timeline', etc.
agGridOnReady EventEmitter - Event emitted when grid is ready
agGridOnFilterApply EventEmitter - Event emitted when filters are applied

Usage Guidelines

  1. Grid Mode: Use 'clientSide' for small datasets, 'serverSide' for large datasets
  2. Column Definitions: Define columns using columnDefs array with headerName, field, and width
  3. Row Data: Provide rowData array for client-side grids
  4. Server-Side: Configure datasource for server-side grids to handle data fetching
  5. Performance: Use pagination, virtual scrolling, and server-side mode for large datasets
  6. Filtering: Grid supports built-in filtering and custom filter components
  7. Sorting: Enable sorting on columns using sortable property in columnDefs
  8. Accessibility: Ensure grid has proper ARIA labels and keyboard navigation
