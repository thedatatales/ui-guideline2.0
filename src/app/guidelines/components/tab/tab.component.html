Tab Component

The tab component organizes content into multiple panels accessible via tabs. It supports horizontal and vertical layouts, icons, routing, and various styling options.

Component Properties

Property Type Default Description
bntv-tabs
selectedIndex number 0 Index of the selected tab - use two-way binding [(selectedIndex)]
theme string 'accent' Theme color: 'primary', 'accent', 'warn'
isVertical boolean false Display tabs vertically
strechedTabs boolean false Stretch tabs to fill available width
enableRouting boolean false Enable routing for tabs
headerPosition string 'above' Position of tab header: 'above' or 'below'
selectedIndexChange EventEmitter - Event emitted when selected tab index changes
bntv-tab
title string - Title text displayed on the tab
icon string - Icon for the tab (format: 'iconName$fontset$fontSetName')
isDisabled boolean false Disable the tab
route string - Route path (used when enableRouting is true)
badge string - Badge text to display on the tab
hide boolean false Hide the tab

Usage Guidelines

  1. Tab Organization: Use tabs to organize related content into logical groups
  2. Tab Count: Limit the number of tabs to avoid overwhelming users (typically 3-7 tabs)
  3. Icons: Use icons to enhance visual recognition and improve navigation
  4. Vertical Layout: Use vertical tabs when horizontal space is limited
  5. Routing: Enable routing for tabs that represent different pages or views
  6. Accessibility: Ensure tabs have proper keyboard navigation and ARIA labels
  7. Content Loading: Consider lazy loading tab content for better performance
