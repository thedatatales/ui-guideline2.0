Tree Component

The tree component displays hierarchical data structures in a tree format. It supports expandable/collapsible nodes, custom templates, and nested data structures.

Basic Tree

Basic tree displays hierarchical data with expandable nodes. The treeData property contains the hierarchical structure, and nodeTemplate defines how each node is rendered.

Component Properties

Property Type Default Description
treeData ITree[] - The hierarchical data to display in the tree
nodeTemplate TemplateRef<any> - Template for rendering each tree node
hasChildFun (index: number, node: ITreeNode) => boolean - Function to determine if a node has children
transformerFun (node: ITree, level: number) => any - Function to transform tree nodes
treeNodePaddingIndent string '40' Padding indent for tree nodes
hideToggleButton boolean false Flag to show or hide the toggle icon
allowClickWithoutToggle boolean false Allow clicking nodes without toggling expansion
nodeToggle EventEmitter - Event emitted when a node is toggled
loadMore EventEmitter - Event emitted when "load more" is clicked

Usage Guidelines

  • Hierarchical Data: Use tree component to display file systems, organizational structures, or any hierarchical data
  • Custom Templates: Use nodeTemplate to customize the appearance and behavior of tree nodes
  • Expandable Nodes: Nodes with children are automatically expandable/collapsible
  • Nested Structures: Tree component supports deeply nested data structures
  • Load More: Implement load more functionality for large datasets
  • Node Selection: Add click handlers in custom templates to handle node selection
  • Styling: Customize node appearance using CSS classes in your templates
