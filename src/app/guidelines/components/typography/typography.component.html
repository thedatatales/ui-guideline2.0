Typography Guidelines

A comprehensive guide to using typography in the Design System

1. Typeface

Primary Font

Poppins (sans-serif)

Used across the product for clarity, consistency, and accessibility.

Font Stack

font-family: var(--fontFamily);

Default value: 'Poppins'

Rationale

  • Optimized for UI readability
  • Excellent performance at small sizes
  • Industry-standard choice for modern applications

3. Hierarchy Rules

Typography hierarchy follows this order of priority:

  1. Font size (primary) - Most important for establishing hierarchy
  2. Font weight (secondary) - Adds emphasis and structure
  3. Color (tertiary) - Provides additional visual distinction
  4. Spacing - Grouping and rhythm for readability
Note: Do not rely on color alone to convey hierarchy or meaning.

4. Line Height & Readability

Headings: 1.4-1.5x font size
Body text: 1.4-1.5x font size
Small text: Never below 1.4x

These ratios improve readability and visual flow. Proper line height ensures text is comfortable to read and maintains visual rhythm.

5. Color Usage & Accessibility

Token Usage WCAG AA
--grayColor800 Primary, body text, headings Pass
--grayColor700 Secondary text Pass
--grayColor600 Supporting info, captions Pass
--grayColor500 Disabled text, metadata Limited

Accessibility Rules

  • --grayColor500 and lighter must never be used for critical information
  • Long-form text must use --grayColor800 or --grayColor700
  • Minimum contrast ratios:
    • Normal text: 4.5:1
    • Large text (18px+): 3:1

6. Spacing & Rhythm

Heading + content: 0.5-1.5x line height
Paragraph spacing: 1x line height
Section spacing: 2-3x body line height

Consistent spacing is essential for typographic rhythm. Use spacing tokens to maintain visual consistency across the application.

7. Layout Guidelines

  • Text alignment: Left-aligned for body text and paragraphs
  • Optimal line length: 45-75 characters for optimal readability
  • Max content width: 600-720px for reading comfort
  • Avoid: Center-aligned text for paragraphs and long-form content

8. Accessibility Compliance

This typography system:

  • Meets WCAG 2.1 AA standards
  • Avoids low-contrast text for primary content
  • Uses minimum 14px body text
  • Supports screen readers with semantic HTML (h1-h6, p)

9. Do & Don't

Do

  • Use defined styles only (h1-h6, .body-1, .body-2, .body-3)
  • Maintain consistent hierarchy
  • Use proper semantic HTML (h1-h6, p)
  • Follow spacing guidelines for rhythm

Don't

  • Introduce new font sizes without approval
  • Use light colors (--grayColor500 or lighter) for important content
  • Use multiple fonts unnecessarily
  • Center-align long paragraphs
