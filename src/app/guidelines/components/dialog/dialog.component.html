Dialog Component

The dialog component provides modal dialog windows using GenericModalService. It supports opening dialogs with components or templates, confirmation dialogs, and various configuration options.

Dialog with Component

Dialogs can be opened with Angular components. Use GenericModalService.openDialog() method and pass the component class along with configuration options.

Confirmation Dialog

Use openDialogConfirmation() method to display confirmation or prompt dialogs. This method returns an Observable that emits 'OK' or 'CANCEL' based on user action.

Service Methods

Method Parameters Returns Description
openDialog() componentOrTemplateRef, config? MatDialogRef Opens a dialog with a component or template
openDialogConfirmation() message?, confirmText?, cancelText?, dialogHeader? Observable<'OK' | 'CANCEL'> Opens a confirmation/prompt dialog

Dialog Configuration Options

Property Type Default Description
width string - Width of the dialog (e.g., '500px', '50%')
height string - Height of the dialog
hasBackdrop boolean true Whether the dialog has a backdrop
disableClose boolean false Whether the dialog can be closed by clicking outside or pressing ESC
autoFocus boolean true Whether the dialog should focus the first focusable element
data any - Data to pass to the dialog component (accessible via MAT_DIALOG_DATA)
maxWidth string - Maximum width of the dialog
minWidth string - Minimum width of the dialog

Usage Guidelines

  • Component Dialogs: Use component dialogs for complex content that requires its own logic and state
  • Template Dialogs: Use template dialogs for simple content that doesn't need separate component logic
  • Confirmation Dialogs: Use confirmation dialogs for user confirmations before performing critical actions
  • Data Passing: Pass data to dialog components using the data property in configuration
  • Dialog Results: Subscribe to afterClosed() observable to handle dialog results
  • Backdrop: Set hasBackdrop to false for non-modal dialogs that allow interaction with background
  • Close Behavior: Use disableClose to prevent accidental closing of important dialogs
  • Responsive Design: Use percentage-based widths for responsive dialogs
