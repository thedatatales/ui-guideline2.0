Selectbox Component

Component Properties

Property Type Default Description
label string - Label text for the selectbox
placeholder string 'Please Select' Placeholder text when no option is selected
itemList any[] - Array of options to display
displayExpression string - Expression to display option text (e.g., '$name')
value string - Key name for the option value
data any - Selected value(s) - use two-way binding [(data)]
isMultipleSelect boolean false Enable multiple selection
disabled boolean false Disable the selectbox
required boolean false Make the field required
iconName string - Icon name to display
iconPosition string - Icon position: 'left' or 'right'
appearance string 'outline' Form field appearance: 'outline', 'fill', 'legacy', 'standard'
search boolean false Enable search functionality in dropdown
trackByKey string - Key for tracking items (improves performance with large lists)
uniqueItemKey string - Unique key property name for items (used for optimization)
displaySelectedCount number - Display selected count when number of selections exceeds this value (multi-select)
triggerTemplate string - Template type for selected items display: 'chip' (multi-select)
optionDisabledFn Function - Function to determine if an option should be disabled: (option: any) => boolean
isDefaultOption boolean false Show default "Select All" option (multi-select)
dynamicClass string - Additional CSS class to apply dynamically
infoMessage string - Info message text displayed with info icon
infoMessageTemplateRef TemplateRef - Template reference for custom info message

Usage Guidelines

  1. Option Lists: Provide clear, descriptive option labels
  2. Display Expression: Use displayExpression to format complex option displays
  3. Multi-Select: Use multi-select for scenarios where users need to choose multiple options
  4. Search: Selectbox includes built-in search functionality for long lists
  5. Validation: Use required property with form validation for mandatory fields
  6. Accessibility: Ensure selectbox has proper labels and keyboard navigation
  7. Performance: For large lists, consider using dynamic mode or pagination
