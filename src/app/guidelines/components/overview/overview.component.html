UI Guidelines – Overview

Legacy BNtv Component Library

Purpose

This guide defines how to use legacy BNtv UI components consistently and correctly across applications.

It exists to:

  • Maintain visual and behavioral consistency
  • Prevent ad-hoc styling and custom overrides
  • Reduce UI regressions in legacy systems
  • Provide a shared reference for teams working on BNtv-based UIs

What This Guide Covers

Foundations

  • Color palette
  • Typography rules
  • Design tokens and variables

Component Guidelines

  • Component purpose and intent
  • Supported variants and states
  • Correct usage patterns
  • Accessibility considerations

Best Practices

  • Do's and don'ts
  • Common mistakes to avoid
  • Consistency rules for implementation

What This Guide Does Not Cover

  • New or experimental UI components
  • Modern design-system or token-first architectures
  • Product-specific UX decisions

This guide is strictly for existing BNtv components and their correct usage.

Who This Is For

  • Frontend Developers implementing BNtv components
  • Designers working within legacy UI constraints
  • Tech Leads & Reviewers enforcing UI consistency
  • QA Teams validating expected UI behavior

How to Use This Guide

  1. Start with Color Palette and Typography to understand foundations
  2. Navigate components by category from the left menu
  3. Use documented examples instead of custom implementations
  4. Refer to usage guidelines before extending or modifying behavior

Guiding Principle

Use components as documented.
If something isn't documented, assume it isn't supported.

Consistency over customization.
